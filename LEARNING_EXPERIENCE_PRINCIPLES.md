## North star (`EP-029`)

> **What should I learn next?**

Not “browse documentation.” Not “watch everything.” Help the customer choose the **next calm step**.

## How people learn here

1. **Just-in-time** — short lessons near the moment of need.  
2. **Journey-staged** — Getting Started → Money concepts → Security → Records → Sharing.  
3. **Glossary-aligned** — same words as money UX (`GLOSSARY.md`, microcopy).  
4. **Optional** — never block deposits/withdrawals behind learning (`EDU-006`).  
5. **Peaceful** — no streaks, points, certificates, or public leaderboards.

## Progressive disclosure

| Layer | Behavior |
| --- | --- |
| Learn home | One **recommended next lesson** + path overview |
| Path | Lessons in order; skip allowed |
| Lesson | Problem → definition → what you’ll see → next step |
| Deep links | From Help, Success Hub, empty states — never forced modals |

## Reading levels

- Default: clear adult prose, short paragraphs.  
- Prefer concrete app vocabulary over abstract finance theory.  
- One primary idea per lesson.  
- Avoid denser than Help Center articles already approved.

## Financial literacy

Teach **platform-honest** literacy:

- Accrued ≠ Credited ≠ Available ≠ Withdrawable  
- Pending and review are normal, not failures  
- Statements project ledger truth  
- Referrals are privacy-first  

Never teach guaranteed returns, risk-free myths, or legal conclusions.

## Completion philosophy

- Completion means “I read this” — not mastery certification.  
- Progress is private and calm (Completed / In progress / Recommended).  
- Incomplete is fine forever.  
- No grades, quizzes, or time pressure.  
- Marking complete is optional and reversible in spirit (re-open anytime).

## Search

- Search across titles, summaries, bodies, and path names.  
- Empty search shows catalog / recommended.  
- Empty results: honest + link to Help / Support.

## Video usage

- Video is optional amplification — never required.  
- Prefer short, captioned, same calm voice.  
- G3 may ship without hosted video; principles still apply when video arrives.  
- Autoplay off. Transcript/summary always available as text lesson.

## Article / lesson standards

Every lesson includes:

1. Title answering a customer question  
2. Estimated time (honest minutes)  
3. Short summary  
4. Body (glossary-safe)  
5. Related lessons  
6. Optional deep link into the app  
7. Footer: guidance, not advice  

## Empty states

| State | Intent |
| --- | --- |
| No progress yet | “Start with the recommended lesson — or browse a path.” |
| Path complete | Quiet acknowledgement; suggest another path — no confetti |
| Search miss | Retry + Help Center |
| Lesson missing | Not found → Learn home |

## Recommendation logic (“Next best lesson”)

1. Prefer the first **incomplete** lesson in the active recommended order.  
2. Order is product-defined by journey (Getting Started first).  
3. If all done: recommend a refresh lesson or Help — never invent urgency.  
4. Never recommend based on FOMO deposit pressure.  
5. Recommendations may later use wallet/investment state lightly (e.g. “Withdrawals in review” lesson when applicable) — still glossary-honest, never engine math.

## Explicit non-goals

- LMS platforms  
- Quizzes / certificates / CE credits  
- Points, badges-as-economy, streaks  
- Mandatory courses before money actions  
- SEO magazine / marketing content mills
